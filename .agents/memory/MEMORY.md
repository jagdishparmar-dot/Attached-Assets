- [Coldverse project](coldverse-project.md) — DMS with Expo Driver App (mobile) + React/Vite Admin Panel (admin) + Express API server; DB provisioned with 5 tables; OTP removed from driver app.
- [Orval hooks options shape](orval-hooks-options.md) — generated hooks don't accept bare `{ refetchInterval }` without `queryKey`; omit or pass full shape.
- [expo-location accuracy null](expo-location-accuracy.md) — `LocationCoords.accuracy` is `number | null`; always use `?? 0` when assigning to a `number` field.
- [Orval inline request bodies](orval-inline-request-body.md) — openapi write-endpoint bodies must use a named `$ref` component, not inline schema, or codegen emits duplicate exports (TS2308).
- [Driver ↔ Staff mirroring](driver-staff-mirror.md) — drivers & staff are separate tables; POST/PATCH /drivers mirrors into staffTable (role=driver) in a txn so Operations drivers also show under Workforce.
- [Delivery number generation](delivery-number-generation.md) — CV-DEL number is the only unique col, generated by scanning max+1; needs per-row txn + retry-on-23505 under concurrency.
- [Mobile backend domain](mobile-backend-domain.md) — release builds need `EXPO_PUBLIC_DOMAIN` set in the EAS production env (the `*.replit.app` deploy host) or they have no backend.
- [Prod incident debugging](prod-incident-debugging.md) — triage "app not working" via healthz + deployment logs + prod read-only SQL before code changes.
